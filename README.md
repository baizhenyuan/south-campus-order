# 南校区点餐小系统

这是一个给手机使用的小网页。她打开主页选择附近店铺和想吃的东西，提交后你在查看页看到订单明细。

## 本机试用

如果电脑已经装了 Node.js，在这个文件夹里运行：

```bash
npm start
```

然后打开：

- 她点餐：http://127.0.0.1:5173/
- 你查看订单：http://127.0.0.1:5173/admin.html

## 公网安全模式

公网部署时请一定设置两个口令：

```bash
ACCESS_KEY=给她的点餐口令 ADMIN_KEY=你的后台口令 npm start
```

部署成功后，把地址这样发给她：

```text
https://你的域名/?key=给她的点餐口令
```

你自己看订单用：

```text
https://你的域名/admin.html?key=你的后台口令
```

不要把 `ACCESS_KEY` 和 `ADMIN_KEY` 设成一样。口令不要写进代码，只放在部署平台的环境变量里。

免费部署时不要添加持久化磁盘，也不要选择付费实例。Render 免费 Web Service 可以公网访问，但会休眠，订单记录可能在服务重启、重新部署或休眠后丢失。

推荐部署到支持 Node.js、HTTPS、环境变量的平台。不要直接把家里电脑路由器端口映射到公网；那样维护防火墙、HTTPS 和系统更新会更麻烦。

上线前检查：

- `ACCESS_KEY` 至少 16 位，发给她。
- `ADMIN_KEY` 至少 16 位，只给你自己。
- 两个口令不要相同。
- 使用 `https://` 链接。
- Render 选择 Free，不添加 Disk。

## 给同一个 Wi-Fi 下的手机使用

启动时让服务监听局域网：

```bash
HOST=0.0.0.0 npm start
```

然后把电脑的局域网 IP 换进地址里，例如：

- 她点餐：http://你的电脑IP:5173/
- 你查看订单：http://你的电脑IP:5173/admin.html

## 给查看页加密码

启动时加上 `ADMIN_KEY`：

```bash
ADMIN_KEY=你的密码 HOST=0.0.0.0 npm start
```

查看页地址就用：

```text
http://你的电脑IP:5173/admin.html?key=你的密码
```

点餐页不需要密码，方便她直接用。

## 修改菜单

菜单在这里：

```text
public/menu.json
```

可以直接增删店铺、菜名、价格、分类。现在内置了南校区小吃街、万达广场、附近饭店、奶茶甜品、蒙餐/清真等候选项。

## 数据位置

收到的订单保存在：

```text
data/orders.json
```
